F — No Se Solicita · Restricciones de Confidencialidad

Lista explícita de lo que el PROVEEDOR no solicita y no requiere para la ejecución del contrato. Para protección mutua.

8 ítems
Explícito
✗

Base de datos de clientes, asegurados o prospectos reales

No se requiere acceso a la base de datos de clientes de Salud S.A. para desarrollar los agentes. Se usarán datos anonimizados/ficticios para pruebas. Los agentes accederán a datos solo a través de HubSpot API con el mínimo permiso requerido.

✗

Contraseñas o credenciales de cuentas de usuario directas

Todas las integraciones se realizan mediante tokens de API, OAuth 2.0 o Service Accounts — mecanismos diseñados para acceso programático seguro sin contraseñas. Nunca se solicitan contraseñas de personas.

✗

Acceso a sistemas internos de gestión de pólizas o reclamaciones

Los agentes operan exclusivamente en el ciclo comercial (ventas y marketing). No se solicita ni se necesita acceso a sistemas de core insurance, gestión de pólizas, reclamaciones médicas ni sistemas administrativos de SALUDSA.

✗

Información médica o de salud de asegurados

En ningún caso el proyecto involucra datos de salud o médicos de ninguna persona. Esto está explícitamente fuera del alcance contractual y sería una violación de la LOPDP.

✗

Estados financieros, presupuestos o datos contables internos de SALUDSA

No son necesarios para el desarrollo de los agentes. Los únicos datos económicos que se manejan son los de campañas de Meta Ads (presupuesto de pauta) que SALUDSA define directamente en su cuenta de Meta Business Manager.

✗

Acceso de Administrador a ninguna plataforma (Meta, HubSpot, Google)

Se solicita solo el rol mínimo requerido: Anunciante en Meta, usuario con permisos limitados en HubSpot, y editor del Sheets específico en Google. Roles de administrador no son necesarios y no se solicitan.

✗

Acceso a la red interna o VPN corporativa de SALUDSA

El proyecto opera en una DMZ agéntica en Azure, separada de la red corporativa. La conectividad se establece mediante reglas de firewall específicas para los endpoints autorizados, no mediante acceso general a la VPN.

✗

Información de empleados, nómina o RRHH de SALUDSA

El único dato de RNs que se solicita es el nombre y número de contacto de los 15 del grupo piloto, única y exclusivamente para configurar el acceso al bot Telegram del Proyecto Marketing. No se solicita ni se requiere ningún otro dato de empleados.
